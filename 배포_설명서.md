# 📅 생활 플래너 — 설치 & 운영 설명서

개인 생활 계획표 시스템. **웹/안드로이드/아이폰 어디서나 같은 데이터**로 쓰고, **카톡/문자/사진/PDF를 AI가 읽어 일정으로 자동 등록**해주는 자급형 플래너다.

> 이 문서는 **사람이 읽어도 되고, Claude(클로드)에게 통째로 주고 "이 설명서대로 내 서버에 설치해줘"라고 시켜도 되도록** 작성했다. 끝부분에 **[Claude에게 시키는 법]** 섹션이 따로 있다.

---

## 1. 이게 뭘 하는 물건인가 (기능)

- **주간 계획이 기본**, 월/년 계획도 가능. 일정 추가·수정·삭제
- **반복 일정**(매일/매주/요일지정/매월/매년), **⭐ 중요 표시**(목록에서 별 탭으로 바로 토글)
- **완료 체크 / 미완료 사유 기록**(프리셋 토글 + 직접입력 → 통계용)
- **오늘 완료율** + **통계**(장소별 달성률, 미완료 사유별 일정 목록)
- **장소별 일일 점검**(집/학원/… 별로 오늘 할 일 점검, 없으면 "없음" 체크)
- **알림(자체 앱 푸시)**: 시작 전 알림(일정별/전역 기본값), 월·금 계획 입력 알람, 빈 요일 아침 알람
- **자동입력(AI 파싱)**: 카톡/문자/메모 텍스트, **사진·PDF(OCR)**, 안드로이드 **카톡 알림 자동읽기**
- **지난 일정 자동 정리**(접어두고 미완료로 보존), **PWA**(홈 화면 설치 시 앱처럼)

### 장소 개념 (중요)
- 장소 = 태그. 기본 6개: `집 / 데까르트 / 수담자 / 화천 / 여주 / 직접입력` (이름은 자유롭게 바꿈)
- 그중 **하나(예: 본인이 운영하는 학원)**가 "자동읽기 대상" → 그 단톡방 카톡이 **검토 대기 큐**로 자동 제안되고, 본인이 **승인**해야 확정
- 나머지 장소는 직접 입력 (특히 아이폰)

## 2. 전체 구조

```
┌──────────────────────────────────────────────┐
│            서버 (Linux + Docker)               │
│  life-planner 컨테이너 (Flask, 내부 5558)      │
│   - REST API + SQLite(일정 DB)                 │
│   - 스케줄러: 알림 발송 (웹푸시 VAPID)         │
│   - Claude CLI: 메시지/사진 → 일정 파싱        │
│            Tailscale Funnel (공개 https)       │
└──────────────────────────────────────────────┘
        ▲              ▲                ▲
     웹(PWA)      안드로이드 앱      아이폰 앱
                  + 카톡 자동읽기    (직접 입력)
              ← 전부 같은 서버·같은 데이터 →
```

핵심: **백엔드 1개 + 클라이언트 여러 개**. 기기마다 따로 만들 필요 없이, 반응형 웹(PWA)을 "홈 화면에 추가"하면 앱처럼 된다.

## 3. 사전 준비물

| 항목 | 설명 | 필수? |
|------|------|-------|
| **Linux 서버** | Ubuntu 등. 24시간 켜져 있고 Docker 설치 가능 | 필수 |
| **Docker** | `curl -fsSL https://get.docker.com \| sudo sh` | 필수 |
| **Tailscale** | 공유기 설정 없이 외부(https) 노출용 | 필수 |
| **프로젝트 파일** | `app.py`, `templates/`, `static/`, `Dockerfile`, `requirements.txt` | 필수 |
| **Claude Code CLI + 로그인** | 자동입력(AI 파싱)용. 없으면 **자동입력만 비활성**(나머지 기능은 정상) | 선택 |

> **자동입력(AI)용 LLM**: Claude **Max 구독**이 있으면 API 키 없이 `claude` CLI로 무료로 쓴다(추가 비용 0). 또는 Anthropic **API 키**를 config에 넣어도 된다(소액 유료).

## 4. 서버 배포 (단계별)

### 4-1. 프로젝트 파일을 서버로
프로젝트 폴더를 서버 `~/life-planner` 에 올린다. (git이 있으면 clone, 없으면 scp/tar로 복사)

### 4-2. 설정 파일 작성 — `~/life-planner/config.json`
```json
{
  "ui_password": "원하는_비밀번호",
  "anthropic_api_key": "",
  "claude_cli_path": "",
  "claude_model": "claude-haiku-4-5-20251001",
  "cli_timeout": 120,
  "timezone": "Asia/Seoul",
  "port": 5558
}
```
- `ui_password`: **공개 주소의 유일한 보안벽이므로 반드시 설정**(길고 강하게)
- `anthropic_api_key`: 비우면 → 서버의 `claude` CLI(Max 구독) 사용. 넣으면 → API 직접 호출
- `secret_key`는 첫 실행 시 자동 생성되어 이 파일에 저장된다(로그인 유지용, 손대지 말 것)

### 4-3. (자동입력 쓸 경우) Claude CLI를 컨테이너에 포함
`Dockerfile`이 이미 **Node.js + `@anthropic-ai/claude-code`**를 설치하도록 되어 있다.
그리고 **Max 로그인 자격증명을 서버로 복사**해 컨테이너에 마운트한다:
```bash
# 로컬(claude 로그인된 PC)에서:
scp ~/.claude/.credentials.json  서버:~/.claude/.credentials.json
```
> 자동입력이 필요 없으면 이 단계와 아래 `-v ~/.claude:/root/.claude` 마운트를 생략해도 된다.

### 4-4. 빌드 & 실행
```bash
cd ~/life-planner
docker build -t life-planner .                 # node+claude 설치라 수 분 걸림
docker rm -f life-planner 2>/dev/null
docker run -d --name life-planner --restart unless-stopped \
  --network host \
  -v ~/life-planner:/app \
  -v ~/.claude:/root/.claude \
  life-planner
curl localhost:5558/api/health                 # {"ok":true,...} 나오면 성공
```
- `--network host`: 5558 포트를 서버에 직접 노출
- `-v ~/life-planner:/app`: DB(`planner.db`)·config 영구 보관
- `-v ~/.claude:/root/.claude`: (자동입력용) claude 로그인 마운트

### 4-5. Tailscale Funnel로 외부 공개
```bash
curl -fsSL https://tailscale.com/install.sh | sudo sh
sudo tailscale up                      # 뜨는 URL로 로그인·기기 승인
sudo tailscale set --operator=$USER    # 이후 sudo 없이 제어
# Funnel은 443 / 8443 / 10000 포트만 지원. 비어있는 포트 선택:
tailscale funnel --bg --https=443 localhost:5558
tailscale funnel status                # https://<호스트>.<tailnet>.ts.net 확인
```
- 이미 다른 서비스가 443을 쓰면 8443이나 10000을 쓴다 (예: `--https=10000`).
- 결과 주소 예: `https://myhost.tailXXXX.ts.net` → **이게 폰에서 접속할 주소**

## 5. 폰에 설치 (PWA)

### 아이폰 (iOS)
1. **Safari로** 접속 (크롬 아님!): `https://<호스트>.ts.net[:포트]`
2. 비밀번호 입력
3. 하단 **공유 버튼(⬆️) → "홈 화면에 추가"**
4. **홈 화면 아이콘으로 실행** (← 이걸로 열어야 알림이 됨)
5. 설정 탭 → **"이 기기에서 알림 받기"** → 허용 → 테스트 알림 확인
> iOS는 **홈 화면 설치(PWA) 후에만** 푸시 알림이 온다. iOS 16.4 이상 필요.

### 안드로이드
1. **Chrome으로** 접속 → 비밀번호 입력
2. **⋮ 메뉴 → "앱 설치"(또는 홈 화면에 추가)**
3. 설정 탭 → "이 기기에서 알림 받기" → 허용
> 안드로이드는 크롬 탭에서도 알림이 오지만, 설치하면 앱처럼 편하다.

> 로그인은 **30일 유지**되어 매번 비번 칠 필요 없다.

## 6. 안드로이드 카톡/문자 → 자동 일정 등록 (선택)

특정 단톡방(예: 학원 공지방) 메시지를 **자동으로 읽어 검토 큐에 제안**하게 만든다.
**무료 앱 LlamaLab Automate**를 사용한다. (아이폰은 OS 제약으로 불가)

### 6-1. 수신 토큰 확인
플래너에 로그인한 상태에서 브라우저로 `…/api/ingest-token` 열면 토큰이 나온다. (또는 Claude에게 물어봐서 settings의 `ingest_token`을 확인)

### 6-2. Automate 플로우 만들기 (자동읽기 폰에서)
1. Play스토어에서 **Automate** (제작사 **LlamaLab**) 설치
2. **폰 설정 → 알림 접근 허용 → Automate 켜기** (필수)
3. 새 Flow 생성 → 블록 3개를 순서대로 연결:
   - **Flow beginning** → **When notification posted** → **HTTP request** → (다시 notification으로 루프)
   - **When notification**: `Posted by application` = **카카오톡**, 출력변수 `message`, `title`
   - **HTTP request**:
     - Request method: **POST**
     - Request URL(따옴표 포함): `"https://<호스트>.ts.net[:포트]/api/ingest?token=토큰"`
     - Body 내용 = `message` 변수 (따옴표 없이)
   - HTTP의 끝(OUT)을 다시 When notification의 IN으로 연결 → 무한 반복
4. **START** 눌러 실행. **배터리 최적화 무시** 체크(절전으로 죽는 것 방지)

### 6-3. 제외할 방/문구
광고·잡담방까지 다 읽으면 검토 큐가 지저분해진다.
플래너 **설정 탭 → "자동읽기 제외"**에 방 이름이나 문구(예: `광고`)를 등록하면, 그 단어가 든 메시지는 무시된다(AI 호출도 안 함).

> 동작 흐름: 카톡 도착 → Automate가 서버로 POST → AI가 일정 추출 → **검토(📥) 탭에 자동 제안** → 승인/거절/수정.

## 7. 자동입력 사용법 (앱 안에서)

**자동입력 탭**에서:
- **텍스트**: 카톡/문자/메모를 붙여넣고 "텍스트 → 일정 변환"
- **사진/PDF**: 시간표·가정통신문 캡처·안내 PDF를 골라 "사진/PDF → 일정 변환" (AI가 OCR로 읽음, 여러 장 가능)
- **출처** 선택: `직접`(바로 등록) / `자동`(검토 큐로)
- **공유로도 등록**: 갤러리/카톡에서 사진·텍스트를 **공유 → 플래너** (안드로이드에서 잘 됨)

"다음주 화요일", "내일 3시" 같은 표현을 실제 날짜·시간으로 환산하고, 글 속 장소명을 자동 인식한다.

## 8. 보안

- 주소는 **공개 인터넷**이다(누구나 주소만 알면 접속 시도 가능). 그래서 **`ui_password`가 유일한 보안벽**.
- 비밀번호는 **길고 강하게** (예: `decartes2026!Xy`). 30일에 한 번만 입력하므로 길어도 부담 없다.
- `config.json`, `planner.db`는 **절대 git/공유에 올리지 않는다**(`.gitignore`에 포함됨).
- 자동읽기 토큰(`ingest_token`)도 외부 노출 금지.

## 9. 자주 겪는 문제 (트러블슈팅)

| 증상 | 원인 / 해결 |
|------|-------------|
| 화면이 옛날 그대로 / 수정이 반영 안 됨 | PWA 캐시. 앱 **완전 종료 후 재실행**, 또는 주소 끝에 `?v=숫자` 붙여 1회 접속. (코드 수정 시 `static/sw.js`의 `CACHE` 버전을 올린다) |
| 푸시 알림이 안 옴 (PC) | Windows 알림 설정에서 브라우저 알림 ON, 집중지원 끄기. 한 번 토글하면 풀리기도 함 |
| 알림 발송이 400으로 실패 | (윈도우 푸시 WNS는 `ttl=0` 거부) webpush에 `ttl` 지정 필요 — 코드에 이미 반영됨 |
| VAPID 키 에러 / 푸시 안 감 | VAPID **개인키는 raw 32바이트(base64url)**로 저장해야 함. PEM이면 깨짐 — 코드에 자동 마이그레이션 있음 |
| 자동입력(사진)이 502 | claude는 **root(컨테이너)에서 `bypassPermissions` 거부**. 이미지 OCR은 `--allowedTools Read`로 호출(코드 반영됨), 텍스트는 권한옵션 없이 호출 |
| 자동입력이 "미설정"이라고 뜸 | 서버에 claude CLI 없음/미로그인. `.credentials.json` 마운트 확인, 또는 `anthropic_api_key` 설정 |
| 컨테이너가 config를 못 읽음 | 편집기가 BOM 추가 → 코드가 `utf-8-sig`로 읽게 되어 있음. 권한/경로 확인 |
| 매번 다시 로그인하라고 함 | `secret_key`가 고정 저장돼야 함(config.json). 재시작마다 풀리면 secret_key 미저장 의심 |
| 일정이 두 개씩 등록 | 저장 버튼 빠른 더블탭 → 코드에 이중 등록 방지 가드 있음(최신 버전) |
| 안드로이드 자동읽기 안 됨 | ① Automate 알림 접근 권한 ② HTTP가 **POST**인지(GET이면 405) ③ 카톡 알림 미리보기 ON ④ 발신은 **다른 계정**이어야 알림이 뜸 |

## 10. 커스터마이징

- **장소 이름 바꾸기**: `app.py`의 `PLACES`와 `AUTO_PLACE` 한 줄 수정 후 재배포. 기존 데이터는 DB의 `plans.place`/`daily_checkin.place` 값을 새 이름으로 UPDATE(마이그레이션)해야 끊기지 않는다.
- **미완료 사유 프리셋**: `app.py`의 `MISS_REASONS` 수정
- **알림 기본값**: 설정 탭에서 조정(서버 재배포 불필요)

## 11. [Claude에게 시키는 법] — 이 부분을 클로드에게 그대로 줘도 됨

> 아래는 같이 일하는 사람이 **자기 서버에 직접 구축**할 때, Claude(클로드)에게 시키는 지시문 예시다. 이 설명서 전체와 프로젝트 파일을 클로드가 접근할 수 있는 곳에 두고 아래처럼 요청하면 된다.

```
이 저장소(life-planner)와 "배포_설명서.md"를 참고해서, 내 리눅스 서버에
생활 플래너를 배포해줘. 순서:

1. 내 서버 접속 정보는 (ssh 별칭 또는 ip/계정)이야. Docker·Tailscale 설치 여부를 먼저 확인해.
2. 프로젝트 파일을 서버 ~/life-planner 로 복사해.
3. config.json을 만들어. ui_password는 "내가_정한_비번"으로. timezone은 Asia/Seoul.
4. (자동입력 쓸 거면) 내 PC의 ~/.claude/.credentials.json을 서버로 복사하고
   docker run에 -v ~/.claude:/root/.claude 마운트를 넣어.
5. docker build → docker run (--network host, --restart unless-stopped, ~/life-planner 마운트).
   curl localhost:5558/api/health 로 확인.
6. Tailscale Funnel로 비어있는 포트(443/8443/10000 중)에 5558을 노출하고,
   접속 주소를 알려줘.
7. 배포 후 /api/health, 로그인, 일정 생성, 알림 설정이 되는지 점검해줘.
8. 장소 이름을 내 상황에 맞게 바꾸고 싶어: (집/회사/…). PLACES·AUTO_PLACE 수정하고
   기존 DB 값도 마이그레이션해줘.

트러블슈팅은 배포_설명서.md의 9번 표를 참고해. 막히면 그 원인부터 확인해줘.
```

**Claude 작업 시 주의 (설명서 9번과 동일):**
- 코드 수정 후엔 `static/sw.js`의 `CACHE` 버전을 올려 캐시를 갱신할 것
- 자동입력 이미지 OCR은 `claude -p ... --allowedTools Read` (root에서 `bypassPermissions` 금지)
- VAPID 개인키는 raw 32바이트로 저장, 웹푸시 `ttl` 지정 필수
- config는 `utf-8-sig`로 읽음, `secret_key`는 자동 생성·고정
